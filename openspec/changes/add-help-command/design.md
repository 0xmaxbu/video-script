## Context

项目是一个 AI 视频生成 CLI 工具，当前 `help` 命令依赖 Commander.js 内置帮助，仅显示命令列表和简短描述。新用户无法理解：

- 完整的工作流程（research → script → screenshot → compose）
- 如何配置 API key 和其他选项
- 典型使用案例和示例

## Goals / Non-Goals

**Goals:**

- 提供清晰的项目级帮助，说明工具用途和工作流程
- 通过 `help` 子命令展示详细使用说明
- 支持中文帮助信息
- 保持向后兼容，不影响现有命令行为

**Non-Goals:**

- 不实现完整的文档系统（仅 CLI 内置帮助）
- 不修改现有命令的实现逻辑
- 不添加交互式向导功能（留作后续 enhancement）

## Decisions

### 1. 使用内置方式增强帮助

**决定**：复用 Commander.js 的 `.help()` 方法，在 CLI 入口文件添加自定义帮助文本。

**理由**：

- 最简单的实现方式，无需新增依赖
- 与 Commander.js 架构一致
- 可通过 `help` 命令和 `--help` 选项触发

**替代方案考虑**：

- 独立 Help 命令类（增加复杂度）
- 外部文档系统（过度设计）

### 2. 帮助内容结构

帮助信息包含：

1. 项目简介（一句话描述）
2. 工作流程图示
3. 快速开始命令
4. 命令列表及说明
5. 配置说明
6. 常见问题

## Risks / Trade-offs

- **风险**：帮助文本可能随项目演进而过时
- **缓解**：将帮助文本集中管理，便于更新
