# Proposal: 修复 Script Agent - 增量场景生成

## 问题描述

Script Agent 在生成脚本时因 LLM 输出不稳定而失败：

1. **多个 JSON 输出**：LLM 在一次输出中生成 2-3 个不同的 JSON 响应
2. **截断**：每个 JSON 都在中途被截断，无法完成
3. **解析失败**：当前 CLI 无法处理这种模式，导致 100% 失败率

### 当前流程

```
Research → Script Agent → 2-3 个截断的 JSON → CLI JSON 解析失败
```

## 解决方案

### 1. CLI JSON 解析器增强

改进 CLI JSON 提取以处理多个 LLM 输出：

- 按 Markdown 代码块分割
- 使用括号计数法提取所有完整的 JSON 候选
- 按完整性评分（场景数量、结构有效性）
- 逐个尝试解析每个候选直到成功

### 2. Agent Prompt 策略：增量场景生成

**变更前**：一次 LLM 调用输出完整脚本  
**变更后**：通过顺序工具调用每次输出一个场景

**优势**：

- 每次 LLM 调用输出更小、更完整的 JSON（一个场景）
- 有限输出无截断风险
- 每个场景更容易验证
- 更好的错误恢复（一个场景失败，而非整个脚本失败）

## 范围

### 包含

- [x] CLI JSON 解析器改进（多候选提取）
- [x] Agent prompt 重构以支持增量场景输出
- [x] 带重试逻辑的场景级错误处理
- [x] 失败时部分脚本保存

### 排除（MVP 阶段）

- [ ] 流式 UI 反馈
- [ ] 生成中途场景编辑/纠正
- [ ] 并行场景生成

## 成功标准

1. 脚本生成成功率 >90%
2. 每个场景单独验证
3. 失败的场景可以单独重试而无需重新生成整个脚本
4. 向后兼容现有 research.json 格式

## 影响

- **修复**：解决阻止 E2E 流程完成的当前阻塞问题
- **改进**：更健壮的 LLM 交互，提升开发者体验
- ** Enables**：未来的流式 UI 和渐进式渲染
