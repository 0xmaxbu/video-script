<implementation_notes>
## Key Implementation Details

### Why 45 frames for intro/outro?
Intro and outro are emotional beats that deserve longer transitions (1.5s at 30fps) for dramatic effect. Feature and code scenes are content-heavy and need snappier transitions (1s) to maintain pace.

### Why alternate slide direction?
Alternating left/right creates visual rhythm and a "back-and-forth" flow that feels dynamic. Fixed direction would feel monotonous.

### Why skip first/last transitions?
First scene with enter transition would fade from black (awkward for intro). Last scene with exit transition would fade to black (outro already has its own ending). Direct appearance/end is cleaner.

### Why use linearTiming instead of springTiming?
Transitions should be deterministic and smooth. Spring physics would add unnecessary bounce/wobble to scene changes. Linear timing provides consistent, predictable transitions.

### Common Pitfalls to Avoid
- **DO NOT** use CSS `transition` property - it's non-deterministic in Remotion
- **DO NOT** forget `extrapolateRight: "clamp"` for any `interpolate` calls (not applicable here, but good to remember)
- **DO NOT** add transition after last scene - video should end directly
- **DO NOT** use fixed slide direction - must alternate based on index
</implementation_notes>
