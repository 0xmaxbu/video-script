# Phase 5-2: Resolution Configuration and Annotation Integration

## Goal

Make resolution configurable (for dual 16:9 + 9:16 output per D-01) and ensure annotations render in generated projects.

## Context

From research:
- `remotion-project-generator.ts` hardcodes `resolution: "1920x1080"` in `videoConfig`
- `Root.tsx` hardcodes `width={1920} height={1080}` in Composition
- Generated `Scene.tsx` doesn't import AnnotationRenderer
- `Composition.tsx` doesn't pass annotations to Scene

## Requirements

- COMP-01: Final video matches visual plan - all annotations render
- COMP-02: Video quality feels polished and professional

## must_haves

- Width and height are configurable in GenerateProjectInput (with defaults 1920x1080)
- Generated videoConfig uses dynamic resolution from input
- AnnotationRenderer is imported and rendered in generated Scene.tsx
- Composition passes annotations data to Scene
- VideoPortrait composition is registered in generated Root.tsx

### Task 3: Update Root.tsx composition to use configurable width/height

<read_first>
packages/renderer/src/remotion/Root.tsx
</read_first>

<action>
In `packages/renderer/src/remotion/Root.tsx`, update the Composition component to use configurable width/height via Remotion's `registerConfiguration`. The mechanism:

1. Create a `videoConfig` object with `width` and `height` properties
2. Use `registerConfiguration` from @remotion/cli/config to set defaults
3. Reference these config values in the Composition's width/height props

This allows dynamic resolution without recompiling the Remotion bundle.

Example pattern:
```typescript
import { registerConfiguration } from "@remotion/cli/config";

registerConfiguration({
  video: {
    width: 1920,
    height: 1080,
  },
});
```

The Composition should still have 1920/1080 as hardcoded defaults for backward compatibility when config is not set.

Note: This file is the SOURCE Remotion root that gets bundled, not the generated one. The generated Root.tsx is created by remotion-project-generator.ts. So we need to update both:
1. The source Root.tsx (for direct rendering use)
2. The generated Root.tsx template in remotion-project-generator.ts
</action>

<acceptance_criteria>
- [ ] `grep -n 'width={1920}' packages/renderer/src/remotion/Root.tsx` still matches (backward compatible default)
- [ ] `grep -n 'height={1080}' packages/renderer/src/remotion/Root.tsx` still matches (backward compatible default)
- [ ] `grep -n 'registerConfiguration' packages/renderer/src/remotion/Root.tsx` returns a match (config mechanism)

### Task 4: Update generated Root.tsx to use configurable resolution

<read_first>
packages/renderer/src/remotion-project-generator.ts (lines 215-260, rootContent template)
</read_first>

<action>
In `remotion-project-generator.ts`, update the `rootContent` template string to use `validated.width` and `validated.height` instead of hardcoded 1920 and 1080.

In the Composition component in rootContent, change:
```
width={1920}
height={1080}
```
To:
```
width={${validated.width}}
height={${validated.height}}
```

Note: In the JavaScript template literal, you write `${validated.width}` directly without escaping. When this template is evaluated, it produces actual JavaScript code like `width={1920}` or `width={1080}`.
</action>

<acceptance_criteria>
- [ ] `grep -E 'width={\$\{validated.width\}}' packages/renderer/src/remotion-project-generator.ts` returns a match
- [ ] `grep -E 'height={\$\{validated.height\}}' packages/renderer/src/remotion-project-generator.ts` returns a match

Note: The grep patterns use `-E` for extended regex where `$` and `{` are metacharacters. The pattern `{\$\{validated.width\}}` matches the literal string `${validated.width}` in the template string.
</acceptance_criteria>
