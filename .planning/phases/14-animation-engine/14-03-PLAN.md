  <action>
    Create `packages/renderer/src/remotion/components/KineticSubtitle.tsx` with the following implementation:

    **Props interface:**
    ```typescript
    interface KineticSubtitleProps {
      text: string;
      /** Optional word-level timestamps from TTS (ElevenLabs format). Falls back to even timing. */
      wordTimestamps?: Array<{ word: string; start: number; end: number }>;
    }
    ```

    **Component logic:**
    1. Split text into words using `text.split(/\s+/)`.
    2. Calculate which word is currently "active" based on frame progress:
       - If `wordTimestamps` provided: find the word whose time range contains `frame / fps`.
       - If no timestamps (fallback): evenly distribute words across scene duration. Active word index = `Math.floor((frame / durationInFrames) * words.length)`.
    3. Render each word as a `<span>` with conditional styling:
       - **Active word:** `color: "white"`, `backgroundColor: "rgba(255, 215, 0, 0.3)"`, `borderRadius: "4px"`, `padding: "2px 6px"` — yellow rounded-rect highlight per D-16.
       - **Inactive words (already shown):** `color: "rgba(255, 255, 255, 0.5)"` — dimmed.
       - **Future words (not yet shown):** `color: "rgba(255, 255, 255, 0.25)"` — more dimmed.

    **Container styling (AI Jason style per D-16):**
    ```
    position: "absolute",
    bottom: 80,
    left: "50%",
    transform: "translateX(-50%)",
    backgroundColor: "rgba(0, 0, 0, 0.8)",
    padding: "16px 32px",
    borderRadius: "12px",
    maxWidth: "80%",
    display: "flex",
    flexWrap: "wrap",
    justifyContent: "center",
    gap: "4px 2px",
    ```

    **Word styling:**
    ```
    fontSize: 36,
    fontFamily: "sans-serif",
    color: "white",
    transition: "color 0.1s", // CSS transition OK here — this is for intra-frame smoothing, not Remotion animation
    ```
    **IMPORTANT:** The color transitions between words are driven by `useCurrentFrame()` + conditional rendering, NOT CSS animations. The CSS `transition` property just smooths the frame-to-frame color change during playback. This is acceptable because Remotion renders each frame discretely — the CSS transition won't cause non-deterministic behavior since we're setting the exact color value per frame.

    **Export:**
    ```typescript
    export const KineticSubtitle: React.FC<KineticSubtitleProps>;
    ```

    **Edge cases:**
    - Empty text → return null
    - Single word → that word is always active
    - Very long text → maxWidth: 80% with flexWrap handles overflow
